## Security blocker resolution

The prior five failures were not all production vulnerabilities. The stale ReleaseTruth assertion expected `chain.processBlock` directly inside `PluginProcessor::processBlock`, but production now correctly delegates through `processSpan`. Three tests asserted generic XML hardening absent from stock JUCE 8.0.12; SendBloom now bounds state to 4 MiB and rejects all DTD/entity declarations before host state or preset parsing. The GZIP test incorrectly treated JUCE's declared length metadata as a decompression cap; SendBloom does not accept GZIP/Zip release input, so the corrected test records JUCE semantics without claiming nonexistent protection.
