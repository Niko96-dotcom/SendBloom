# Plan 27-03: Run RC evidence gates and decide promotion truthfully

## Objective

Run every available local automated gate from a clean build directory, collect hashes/logs for artifacts actually produced, probe AU/pluginval capabilities, and produce a release report whose status reflects missing external evidence.

## Tasks

1. Configure/build/test Release from a fresh directory and run `verify-v1.sh` with available pluginval.
2. Discover and validate the actual AU type when local tools/artifacts permit.
3. Package/hash only locally verified artifacts and record exact commands/results.
4. Produce canonical verification; do not tag if CI, DAW, license entitlement, signing/notarization, cleanliness, or other required evidence is absent.

## Verification

- Automated evidence report names exact build path, test count, pluginval/AU result, artifacts, and hashes.
- Release evidence enumerates each REL requirement as verified, failed, or `human_needed`.
- `v1.0.0-rc0` is absent unless every acceptance condition is proven.
