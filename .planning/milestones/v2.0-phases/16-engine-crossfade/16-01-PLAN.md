  <behavior>
    - prepare(sampleRate) configures juce::SmoothedValue ramp duration between 0.02f and 0.05f seconds (default 0.035f)
    - beginCrossfadeTowardFixed() ramps mix gain from 0 (host) to 1 (ProperSRC); beginCrossfadeTowardHost() ramps 1 to 0
    - mixWetBlock applies equal-power sin/cos blend: hostGain = cos(g * halfPi), fixedGain = sin(g * halfPi)
    - Single mid-buffer toggle on 1 kHz sine wet buffer yields maxAdjacentDelta below 0.25f
    - Fade completes in 20–50 ms worth of samples at 48 kHz (960–2400 samples)
  </behavior>
  <action>
Create source/EngineCrossfade.h as a header-only helper struct (namespace sendbloom) patterned on BypassCrossfade but for dual wet paths.

Members: juce::SmoothedValue<float> mixGain (0 = host wet, 1 = fixed/ProperSRC wet), bool crossfading flag, bool targetIsFixed member for post-fade routing.

Constants: static constexpr float kDefaultFadeSeconds = 0.035f; static constexpr float kMinFadeSeconds = 0.02f; static constexpr float kMaxFadeSeconds = 0.05f.

prepare(double sampleRate): reset mixGain with kDefaultFadeSeconds clamped to [kMinFadeSeconds, kMaxFadeSeconds].

beginCrossfadeTowardFixed(): set crossfading true, targetIsFixed true, set mixGain current from last completed state, target 1.0f.

beginCrossfadeTowardHost(): set crossfading true, targetIsFixed false, target 0.0f.

mixWetBlock(const float* hostWet, const float* fixedWet, float* output, int numSamples): per sample call mixGain.getNextValue(), compute equal-power gains, write blended output; when mixGain.isSmoothing() becomes false set crossfading false.

isCrossfading() const noexcept returns crossfading || mixGain.isSmoothing().

targetIsFixedEngine() const noexcept returns targetIsFixed (for idle-engine reset after completion).

Do not repurpose SmoothedParameterBank authenticColorTarget as engine mix control — separate smoother only.

Create tests/EngineCrossfadeTest.cpp with Catch2 tags [verb][EngineCrossfade][XFADE-01].

Case 1 — fade duration: prepare 48000 Hz; beginCrossfadeTowardFixed from host; count samples until !isCrossfading(); REQUIRE count between 960 and 2400 inclusive.

Case 2 — click metric: 512-sample 1 kHz sine at amplitude 0.5 in host and fixed scratch; toggle beginCrossfadeTowardFixed at sample 256; mix full block; compute max adjacent abs delta on output; REQUIRE below 0.25f.

Case 3 — normalized click: same as Case 2 but divide delta by max(peak, 1e-6f); REQUIRE normalized delta below 0.5f.

No heap allocation in mixWetBlock body (stack/local only).
