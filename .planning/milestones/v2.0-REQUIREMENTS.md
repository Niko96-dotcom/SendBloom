# Requirements: SendBloom

**Defined:** 2026-07-06
**Core Value:** The dry guitar stays clean while the wet path delivers gated, distorted reverb that chops hard when you stop playing.

## v2.0 Requirements — Proper 32k SRC (Option C)

Requirements for milestone v2.0. Each maps to roadmap phases 11–18.

### RC1 Safety

- [x] **SAFE-01**: `authentic_color` (32k Color) defaults to off in parameter layout and factory presets
- [x] **SAFE-02**: All 8 factory presets ship with `authentic_color=0` until enablement phase passes acceptance gates
- [x] **SAFE-03**: Production path uses host-rate Schroeder tank; broken accumulator path is not user-facing default

### Core Extraction

- [x] **CORE-01**: `SchroederTankCore` runs at a single `processingRate` with no `hostRate` or `useAuthenticPath` branches inside
- [x] **CORE-02**: Fixed 32,768 Hz core uses unscaled delay table from `SchroederTank32DelayTable`
- [x] **CORE-03**: `HostRateReverbEngine` wrapper preserves existing host-rate tank behavior for RC1 primary path
- [x] **CORE-04**: RT60 within ±15% at size 0.25, 0.5, 1.0 for both host-rate and fixed-rate cores (impulse test)

### Sample-Rate Conversion

- [x] **SRC-01**: `FixedRateAdapter` performs bandlimited hostRate ↔ 32,768 Hz conversion via r8brain-free-src (MIT)
- [x] **SRC-02**: r8brain resamplers constructed and buffers preallocated in `prepare()` only — zero heap in `processBlock()`
- [x] **SRC-03**: `FixedRateAdapter` supports host rates 44.1, 48, 88.2, and 96 kHz with variable block sizes (1 sample to max block)
- [x] **SRC-04**: Internal `Authentic32Mode` enum: Off / LegacyAccumulator / ProperSRC — diagnostics only, not user-facing
- [x] **SRC-05**: Legacy accumulator path retained internally for A/B regression tests only
- [x] **SRC-06**: ProperSRC reduces 14–15 kHz imaging at least 70% versus LegacyAccumulator on impulse fixture at 48 kHz

### Block Integration

- [x] **INTEG-01**: `IReverbEngine` extended with `processBlock()` for fixed-rate path; `processSample()` retained for host-rate compatibility
- [x] **INTEG-02**: `GatedBloomChain` processes reverb+SRC at block level; gates, pressure send, and wet OD remain at host rate
- [x] **INTEG-03**: Wet overdrive, dry routing, gate behavior, and pressure send are unchanged by this milestone
- [x] **INTEG-04**: No new UI controls exposed; existing 32k Color toggle behavior preserved (off by default)

### Diagnostics

- [x] **DIAG-01**: Three-path render harness compares host-rate, LegacyAccumulator, and ProperSRC paths
- [x] **DIAG-02**: Fixtures: impulse, guitar pluck, 220 Hz decay, 880 Hz decay, swept sine 200–8000 Hz
- [x] **DIAG-03**: Metrics: RMS >10 kHz, RMS >14 kHz, dominant tail peak frequency, narrowband dominance, spectral centroid
- [x] **DIAG-04**: ProperSRC host-rate invariance: 44.1/48/96 kHz renders have broadly similar HF metrics within documented tolerance

### Engine Crossfade

- [x] **XFADE-01**: Toggling `authentic_color` crossfades between engines over 20–50 ms without click above threshold
- [x] **XFADE-02**: 1000-toggle stress test passes: no NaN/Inf, no heap alloc, no delay buffer overrun

### Latency & PDC

- [x] **LAT-01**: SRC round-trip latency measured at 44.1, 48, 88.2, and 96 kHz (not guessed)
- [x] **LAT-02**: ADR-003 documents PDC policy: report latency, internal compensation, or defer ProperSRC to Extended/v2.1
- [x] **LAT-03**: Reported plugin latency matches ADR-003 policy when ProperSRC is active; RC1 with 32k off remains zero reported latency

### Documentation

- [x] **DOC-01**: `docs/architecture/ADR-003-proper-32k-src.md` documents why accumulator/hold is insufficient, SRC architecture, library choice, and latency decision

### Testing & Validation

- [x] **TEST-08**: Fixed32SRC produces finite output for all fixtures and host rates
- [x] **TEST-09**: Fixed32SRC does not allocate in `processBlock()` (realtime stress)
- [x] **TEST-10**: Fixed32SRC reset clears all delay and resampler state
- [x] **TEST-11**: Existing HF ringing regression tests pass with ProperSRC at 48 kHz
- [x] **TEST-12**: pluginval strictness 10 passes after integration
- [x] **TEST-13**: Legal metadata audit passes with r8brain MIT license cited

### Enablement (gated — only after acceptance)

- [x] **ENAB-01**: 32k Color may be enabled by default only after TEST-11, DIAG-04, LAT-02, and XFADE-01 pass
- [x] **ENAB-02**: VERB-05 documentation updated to describe ProperSRC bandlimited bridge, not accumulator stepping
- [x] **ENAB-03**: Anti-image SVF demoted or removed after ProperSRC passes HF acceptance gates

## v1 Requirements

### Scaffold & CI

- [x] **SCAF-01**: Repository initialized from pamplejuce-style scaffold (JUCE 8, CMake, C++20)
- [x] **SCAF-02**: Empty passthrough plugin builds as AU and VST3
- [x] **SCAF-03**: GitHub Actions build matrix (macOS, Windows, Linux) passes on passthrough
- [x] **SCAF-04**: pluginval runs in CI at strictness 5 initially, raised to 10 before release
- [x] **SCAF-05**: Plugin metadata uses SendBloom / NkMo / SbLm codes (placeholders verified before ship)

### Parameter System

- [x] **PARM-01**: All APVTS parameter IDs defined in `ParameterIDs.h` per architecture table
- [x] **PARM-02**: `createParameterLayout()` exposes all public params with correct ranges and defaults
- [x] **PARM-03**: `ParameterSnapshot` created once per `processBlock()`; no APVTS atomics in DSP inner loops
- [x] **PARM-04**: Per-parameter smoothing matches architecture spec (attack/release times)
- [x] **PARM-05**: Curve mappings: size→RT60, distn→pow(2.8), level→sin equal-power, send curves
- [x] **PARM-06**: Bypass with 5 ms clickless crossfade

### Input / Output Stages

- [x] **IO-01**: `InputStage` applies input gain dB with soft clip limiter and 50 ms clip-hold flag for UI (wet path only; dry tap is pre-gain unity)
- [x] **IO-02**: `OutputStage` applies output gain dB trim
- [x] **IO-03**: Mono bus contract: stereo in summed to mono; mono/stereo out as dual-mono unless Extended stereo enabled

### Gate System

- [x] **GATE-01**: `EnvelopeDetector` peak follower with configurable attack/release
- [x] **GATE-02**: `NoiseGate` PreSoft profile: 150 ms release, hum silencer on wet input only
- [x] **GATE-03**: `NoiseGate` PostHard profile: ≤7 ms Authentic release, brutal wet chop keyed from input detector
- [x] **GATE-04**: Gate mode toggle (Pre/Post) selects gate placement in wet path only; dry never gated
- [x] **GATE-05**: `input_threshold_db` controls open threshold with 3 dB hysteresis

### Pressure Send

- [x] **SEND-01**: `PressureSend` returns 1.0 when `send_connected` is false (always-on wet feed)
- [x] **SEND-02**: When connected, maps `send_amount` via smoothstep + Firm/Soft exponent curves
- [x] **SEND-03**: Release to zero does not clear reverb tank buffers; tank energy decays naturally
- [x] **SEND-04**: MIDI CC1 (mod wheel) controls `send_amount` when connected

### Reverb Engine

- [x] **VERB-01**: `SchroederTank32` primary engine: 4 series APF → 4 parallel damped combs → modulated tank AP
- [x] **VERB-02**: Fixed delay lengths at 32,768 Hz per architecture table
- [x] **VERB-03**: Size→RT60 mapping `0.25 + 5.75×size^2.4` with smooth feedback gain updates
- [x] **VERB-04**: Dark mode: 55 ms predelay + darker damping (3200 Hz); Bright: no predelay, 8000 Hz cutoff
- [x] **VERB-05**: `authentic_color` (32k Color) uses bandlimited hostRate ↔ 32,768 Hz SRC via r8brain ProperSRC; SchroederTankCore runs at fixed delay-table lengths with per-comb feedback calibration, damping, and optional 9-bit param quantization — original software, not firmware-derived. Legacy accumulator path retained for diagnostics only.
- [x] **VERB-06**: `Fdn8Reverb` fallback behind same interface for tests and Extended reference
- [x] **VERB-07**: RT60 within ±15% at size 0.25, 0.5, 1.0 (impulse test)

### Wet Overdrive

- [x] **OD-01**: `WetOverdrive` processes reverb output only; fixed asymmetric tanh circuit (no drive knob)
- [x] **OD-02**: `distn` blends clean wet ↔ fully driven wet via `pow(distn, 2.8)`
- [x] **OD-03**: Dry path THD unchanged at distn=1, level=1 (verified by dry-tap test)

### Signal Chain

- [x] **CHN-01**: `GatedBloomChain` wires full topology per architecture pseudocode
- [x] **CHN-02**: `ParallelWetMixer` keeps dry at unity; level controls wet return via sin curve
- [x] **CHN-03**: Post gate keyed from input detector, not wet tail (edited-sample behavior)
- [x] **CHN-04**: Zero reported plugin latency in v1; no lookahead
- [x] **CHN-05**: `processBlock()` realtime-safe: no heap alloc, locks, logging, or UI access after prepare

### User Interface

- [x] **UI-01**: Pedal-style layout: In, Size, Lvl, Distn, Out knobs + Dark + Gate Pre/Post toggles
- [x] **UI-02**: `PressureSendPad` with mouse/touch down-drag-up behavior and visual bloom (UI thread only)
- [x] **UI-03**: Clip LED driven from InputStage clip flag
- [x] **UI-04**: Advanced drawer: Gate Sens, Send Feel, 32k Color; Extended Stereo and Dirt OS shown disabled
- [x] **UI-05**: No graphs, confidence meters, or algorithm language in main UI

### Presets

- [x] **PRST-01**: 8 factory presets as XML resources per architecture list
- [x] **PRST-02**: Preset load/save round-trip tests pass

### Testing & Validation

- [x] **TEST-01**: Parameter curve unit tests (RT60, distn blend, send mapping)
- [x] **TEST-02**: Gate unit tests (pre hum suppression, post ≤15 ms chop, dry passes)
- [x] **TEST-03**: Dry-path clean test at distn=1
- [x] **TEST-04**: Pressure send trail test (tank energy 500 ms after release)
- [x] **TEST-05**: Realtime safety stress test (10,000 blocks, varying sizes, no alloc)
- [x] **TEST-06**: pluginval strictness 10 passes
- [ ] **TEST-07**: Plugin loads in Logic, Cubase, REAPER

### Legal & Metadata

- [x] **LEG-01**: No Rainger / Reverb-X / Igor references in product name, metadata, or presets
- [x] **LEG-02**: Clean-room positioning documented; no EEPROM/bytecode reverse engineering
