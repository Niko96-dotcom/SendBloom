# Roadmap: SendBloom

## Milestones

- ✅ **v1.0 SendBloom** — Phases 1–10 (shipped 2026-07-06)
- 🚧 **v2.0 Proper 32k SRC (Option C)** — Phases 11–18 (in progress)
- **Branch:** `feature/proper-32k-src`

## Overview

**v1.0 (shipped):** Ten vertical phases delivered a loadable AU/VST3 gated-dirty-ambience plugin — routing-first ugly chain, then real IO, gates, SchroederTank32, wet OD, pressure send, UI, presets, and release hardening (103 tests, pluginval 10).

**v2.0 (active):** Eight surgical DSP phases replace the broken accumulator/hold 32k Color path with a bandlimited hostRate ↔ 32,768 Hz SRC bridge around an extracted `SchroederTankCore`. RC1 safety freezes 32k Color off by default; ProperSRC is validated via three-path diagnostics before user enablement. Gates, send, wet OD, and dry routing stay unchanged.

### Phase 12: SchroederTankCore Extraction

**Depends on**: Phase 11
**Requirements**: CORE-01, CORE-02, CORE-03, CORE-04

  4. RT60 within ±15% at size 0.25, 0.5, and 1.0 for both host-rate and fixed-rate cores (impulse test)

### Phase 13: FixedRateAdapter + r8brain

**Goal**: Bandlimited hostRate ↔ 32,768 Hz conversion wraps the fixed-rate core with realtime-safe r8brain resamplers
**Depends on**: Phase 12
**Requirements**: SRC-01, SRC-02, SRC-03, SRC-04, SRC-05, SRC-06, TEST-10
**Success Criteria** (what must be TRUE):

  1. `FixedRateAdapter` performs bandlimited conversion via r8brain-free-src with zero heap allocation in `processBlock()`
  2. Adapter supports host rates 44.1, 48, 88.2, and 96 kHz with variable block sizes (1 sample to max block)
  3. ProperSRC reduces 14–15 kHz imaging at least 70% versus LegacyAccumulator on impulse fixture at 48 kHz
  4. Internal `Authentic32Mode` enum (Off / LegacyAccumulator / ProperSRC) available for diagnostics only
  5. Reset clears all delay lines and resampler state completely

### Phase 15: Three-Path Diagnostics

**Goal**: Engineering can prove ProperSRC architecturally fixes HF imaging before any user-facing 32k Color enablement
**Depends on**: Phase 14
**Requirements**: DIAG-01, DIAG-02, DIAG-03, DIAG-04, TEST-08, TEST-11
**Success Criteria** (what must be TRUE):

  1. Three-path render harness compares host-rate, LegacyAccumulator, and ProperSRC outputs side by side
  2. HF metrics (RMS >10 kHz, RMS >14 kHz, dominant tail peak, narrowband dominance, spectral centroid) quantify imaging across fixture suite
  3. ProperSRC renders at 44.1, 48, and 96 kHz have broadly similar HF metrics within documented tolerance
  4. ProperSRC produces finite output for all fixtures and host rates; existing HF ringing regression passes at 48 kHz

**Goal**: SRC round-trip latency is measured and a documented PDC policy is implemented before user enablement
**Depends on**: Phase 16
**Requirements**: LAT-01, LAT-02, LAT-03, DOC-01
**Success Criteria** (what must be TRUE):
